# 🌳 Cloud Infrastructure & Linux Learning Tree

이 저장소는 클라우드 인프라의 핵심 메커니즘을 시스템적 사고(System Thinking)를 바탕으로 파헤치고, 본질적인 작동 원리와 개념을 구조화하여 기록하는 공간입니다.

---

## 📌 Index: 학습 트리 구조 (Tree Structure)

* **Root: 웹 서비스 배포 매커니즘 이해**
  * **Branch 1: 파일 수송 및 위치 정의**
    * `scp` 명령어를 통한 로컬 환경 ➡️ AWS 원격 가상 컴퓨터 파일 전송
    * 리눅스 경로 표시자의 본질 (`~` 홈 디렉토리 vs `/` 최상위 루트)
    * 경로 시스템의 분류 (절대경로 vs 상대경로)
    * 웹 서버 표준 디렉토리 규격 (`/var/www/html/`)
  * **Branch 2: 웹 서버 구동 및 보안 권한 통제**
    * 아파치 웹 서버 요원(`httpd`) 설치 및 활성화
    * 파일 접근 제어 자물쇠 설정 (`chmod 644` 수학적 조합)
    * 소유권 등기 이전 명의 변경 (`chown apache:apache`)
  * **Branch 3: 커맨드 라인 인터페이스(CLI) 시스템 이해**
    * Windows 환경의 마법 지팡이: **파워쉘 (PowerShell)**
    * 시스템 원격 제어의 표준: **리눅스 터미널 (Terminal / Bash)**
    * 오픈소스 생태계의 절대강자: **우분투 (Ubuntu)**

## 📑 Core Essence: 개념별 심층 요약

### 1. 파일 이사와 웹 서버 표준 경로

#### 🛸 파일 수송선 (`scp`)
내 로컬 컴퓨터에 존재하는 웹 사이트 핵심 파일 3대장(`index.html`, `style.css`, `app.js`)을 마스터키(`*.pem` 키페어 파일)를 사용하여 안전하게 AWS 가상 컴퓨터(EC2)의 기본 수신 폴더(`~/`)로 원격 복사·전송하는 명령어입니다.

#### 🏠 홈 디렉토리 (`~`)
리눅스라는 공동 아파트 시스템 안에서 내 계정(`ec2-user`)에게만 독점적으로 부여된 **'나만의 개인 독방'**입니다. 시스템 핵심 구역과 완벽히 격리되어 있어, 최고 관리자 서명(`sudo`) 없이도 파일 생성, 수정, 삭제를 가장 자유롭고 안전하게 수행할 수 있는 시작점입니다.
* **절대경로 주소**: `/home/ec2-user`

#### 🌍 세계 공통 웹 진열대 (`/var/www/html/`)
전 세계 모든 웹 서버 프로그램들이 "여기에 인터넷 손님들에게 보여줄 파일들이 정돈되어 있구나"라고 기본값(Default)으로 찾아가도록 약속된 **'인프라 세계의 도로명 주소'**입니다. 
* **`/`** : 리눅스 컴퓨터의 최상위 본부
* **`var` (Variable)** : 로그, 웹 파일 등 데이터가 자주 추가·변경되는 구역
* **`www` (World Wide Web)** : 인터넷 서비스 전용 방
* **`html`** : 진짜 웹사이트 소스 코드가 위치하는 최종 종착지

### 2. 권한(Permission)과 소유권(Ownership) 통제

#### 🔐 보안 자물쇠 비밀번호 (`chmod 644`)
외부 인터넷 손님이 주소를 입력하고 들어왔을 때, 보안 에러(`403 Forbidden`) 없이 웹페이지 화면을 정상적으로 읽어갈 수 있도록 파일의 자물쇠를 정밀 세팅하는 명령입니다. 세 자릿수 숫자는 고정된 서열을 가집니다.

| 서열 구분 | 할당 점수 | 보유 권한 명세 |
| :--- | :---: | :--- |
| **1등석 : 나 (주인)** | **6** | `4(읽기) + 2(쓰기)` ➡️ 파일 수정 및 열람 가능 |
| **2등석 : 우리 팀 (그룹)** | **4** | `4(읽기)` ➡️ 파일 열람만 가능 |
| **3등석 : 남 (인터넷 손님)** | **4** | `4(읽기)` ➡️ 웹브라우저로 화면을 정상 출력하기 위한 필수 권한 |

* **💡 권한 응용 및 오답 노트**:
  * **`664`** : 우리 팀원에게도 나와 동등하게 수정 권한(`쓰기=2`)을 부여하고자 할 때 사용합니다.
  * **`614`** : 팀원에게 실행(`1`) 권한만 부여하고 읽기·쓰기를 차단하는 설정으로, 웹 소스 파일에서는 매우 어색하고 위험한 조합이 됩니다.

#### 🪪 명의 등기 이전 (`chown apache:apache`)
파일을 처음 전송한 계정(`ec2-user`)으로부터, 실제로 1층 로비에서 인터넷 손님을 맞이하고 서빙을 담당할 **아파치 웹 서버 요원(`apache`)에게 파일의 실질적 소유권을 완전히 넘겨주는 명령**입니다. 이 절차를 통해 아파치 요원이 아무런 보안 제약이나 눈치를 보지 않고 소유 자격으로 파일을 손님에게 가장 신속하게 제공할 수 있게 됩니다.

### 3. 인터페이스 시스템 (PowerShell, Terminal, Ubuntu)

#### ⚡ 윈도우의 끝판왕 지팡이, 파워쉘(PowerShell)
윈도우 환경을 마우스 없이 명령어(텍스트)로 프로그래밍하듯 완벽하게 통제하는 초강력 쉘입니다. 진한 파란색 배경에 명령어 창임을 뜻하는 `>_` 로고를 쓰며, 창을 켜면 파워쉘 모드임을 증명하는 `PS` 프롬프트 심볼이 맨 앞에 붙습니다.
* **파워쉘 전용 환경 변수 정석 문법**: `$env:USERPROFILE` (내 컴퓨터 사용자 계정 루트 경로 주소 호출)

#### 🏛️ 실무 명령어 창의 서열
1. **리눅스 터미널 (Bash)** [🥇 압도적 1위] : 전 세계 클라우드(AWS) 서버의 90% 이상이 가볍고 안정적인 리눅스 OS를 채택하고 있으므로, 엔지니어가 실무에서 가장 많이 쓰는 절대 표준입니다.
2. **윈도우 파워쉘 (PowerShell)** [🥈 2위] : 내 노트북에서 원격 서버로 수송선(`scp`)을 띄우거나 내부 스크립트를 제어할 때 사용합니다.
3. **윈도우 명령 프롬프트 (CMD)** [🥉 3위] : 1980년대 구조에 멈춰있는 낡은 도구로, 현대 인프라 실무에서는 거의 은퇴 수순을 밟고 있습니다.

#### 🐧 리눅스 가문의 대장, 우분투(Ubuntu)
리눅스라는 거대한 뿌리에서 갈라져 나온 핵심 OS 종류 중 하나로, 전 세계 개발자들이 가장 사랑하는 시스템입니다. 초보자 친화적인 GUI 환경과 압도적인 구글링 데이터(생태계)를 자랑합니다. 
현재 실습 중인 대기업 중심의 **'아마존 리눅스(RedHat 계열, dnf/yum 사용)'**와 시장을 양분하고 있는 강력한 라이벌이자 형제 가문**'우분투(Debian 계열, apt 사용)'**입니다.
